# Tesseract Baseline

## Как устроен Tesseract внутри

Tesseract — классический OCR-движок, написанный на C++. Он не использует нейросети в привычном смысле — его архитектура основана на последовательной обработке изображения через несколько независимых этапов.

### Общий пайплайн

```
Изображение
    ↓
1. Предобработка (бинаризация, deskew, шумоподавление)
    ↓
2. Layout Analysis — сегментация страницы на блоки
    ↓
3. Сегментация строк и слов
    ↓
4. Распознавание символов (LSTM)
    ↓
5. Постобработка словарём (DAWG)
    ↓
Текст
```

---

### Шаг 1 — Предобработка

Перед любым распознаванием Tesseract приводит изображение к удобному для анализа виду.

**Бинаризация по алгоритму Оцу.** Изображение переводится в чёрно-белое: алгоритм находит порог яркости, который максимально разделяет пиксели на два класса — фон (белый) и текст (чёрный). Всё темнее порога становится 0, всё светлее — 255.

**Deskew — исправление наклона.** Если страница чуть повёрнута, Tesseract вычисляет угол, при котором горизонтальные проекции пикселей наиболее выражены, и поворачивает изображение обратно.

**Шумоподавление.** Морфологические операции (erosion и dilation) убирают мелкие артефакты — одиночные точки, царапины, шум сканера.

Для рукописного текста предобработка часто работает хуже чем для печатного — неравномерное освещение, тени, переменный нажим пера сбивают бинаризацию.

---

### Шаг 2 — Layout Analysis и PSM

Tesseract строит иерархию структур изображения:

```
Страница
  └── Блоки текста
        └── Параграфы
              └── Строки
                    └── Слова
                          └── Символы
```

Алгоритм **Connected Components Analysis** находит связные компоненты тёмных пикселей (потенциальные символы), группирует их в слова по горизонтальным зазорам, слова — в строки по вертикальному перекрытию, строки — в блоки.

**PSM (Page Segmentation Mode)** указывает Tesseract, на каком уровне этой иерархии находится входное изображение:

| PSM | Поведение |
|-----|-----------|
| 6 | Полный блок текста, авто-сегментация строк |
| 7 | Одна строка — не искать ничего кроме неё |
| 8 | Одно слово |
| 13 | Строка без какой-либо сегментации |

В нашем случае используется `--psm 7`, потому что IAM уже нарезан по строкам — Tesseract не должен самостоятельно искать структуру.

---

### Шаг 3 — Распознавание через LSTM

Современный Tesseract (v4+) использует **двунаправленный LSTM** для распознавания текста.

На вход LSTM подаётся не весь символ сразу, а вертикальные полосы изображения строки, скользящие слева направо. Это позволяет модели учитывать контекст — какие символы были до и после текущей позиции.

Выравнивание предсказаний с символами производится через **CTC (Connectionist Temporal Classification)** — алгоритм, не требующий явной разметки позиции каждого символа.

Главная проблема: Tesseract обучался преимущественно на **синтетических и печатных данных**. Его веса плохо генерализуются на рукописный стиль.

---

### Шаг 4 — Постобработка словарём (DAWG)

После LSTM Tesseract прогоняет распознанные символы через **словарный граф DAWG (Directed Acyclic Word Graph)**. Если распознанная последовательность близка к словарному слову — Tesseract исправляет её.

Для рукописного текста это вредит: правильно распознанные редкие слова, имена или аббревиатуры могут быть исправлены на ближайшее словарное слово. Поэтому в нашем скрипте словарь отключён явно:

```
-c load_system_dawg=false -c load_freq_dawg=false
```

## Разбор кода

### Импорты и константы

```python
import csv
import pytesseract
from PIL import Image
from jiwer import cer, wer
from pathlib import Path
```

`pytesseract` — не самостоятельная нейросеть, а **тонкая Python-обёртка над бинарником Tesseract**. При вызове она сохраняет изображение во временный файл, запускает `tesseract` как внешний процесс через `subprocess`, читает результат из файла и возвращает строку. То есть это буквально звонок к C++-программе.

`PIL.Image` — открывает изображения. `jiwer` считает метрики CER и WER через алгоритм редакционного расстояния. `pathlib.Path` позволяет строить пути через `/` вместо `os.path.join`.

```python
DATA_PATH   = Path("iam")
LINES_DIR  = DATA_PATH / "lines"
GT_FILE    = DATA_PATH / "gt" / "lines.txt"
SPLITS_DIR = DATA_PATH / "splits"
OUTPUT_CSV = Path("results") / "tesseract_baseline.csv"

TESS_CONFIG = "--psm 7 -l eng -c load_system_dawg=false -c load_freq_dawg=false"
```

Все пути и настройки вынесены в константы вверху файла — чтобы при изменении структуры папок менять только одно место. `TESS_CONFIG` — строка аргументов, которая передаётся напрямую в бинарник Tesseract.

### `load_gt()` — парсинг ground truth

```python
def load_gt(gt_file: Path) -> dict[str, str]:
```

Читает `iam/gt/lines.txt` и возвращает словарь `{line_id: транскрипция}`.

Формат строки в файле:
```
a01-000u-00 ok 154 19 408 746 15 87 A|MOVE|to|stop
```

Поля по порядку: идентификатор строки, статус (ok/err), технические метаданные (позиция, размер), затем текст. Слова в тексте разделены символом `|`.

```python
parts = line.split(" ")
line_id = parts[0]        # "a01-000u-00"
status  = parts[1]        # "ok" или "err"
if status != "ok":
    continue              # бракованные строки пропускаем
transcription = " ".join(parts[8:]).replace("|", " ")
```

`parts[8:]` — список слов начиная с 9-го поля. `" ".join(...)` склеивает их через пробел. `.replace("|", " ")` заменяет разделитель IAM на обычный пробел.

### `load_split_ids()` — загрузка сплита

```python
def load_split_ids(splits_dir: Path, split: str) -> list[str]:
```

Читает один из файлов сплита и возвращает список идентификаторов строк. Файлы сплита содержат просто список `line_id`, по одному на строку — они определяют, какие строки входят в train, val и test.

IAM предоставляет два файла валидации (`validationset1.txt` и `validationset2.txt`). Используем первый.

### `line_id_to_image_path()` — построение пути к картинке

```python
def line_id_to_image_path(line_id: str, lines_dir: Path) -> Path:
```

IAM хранит изображения в трёхуровневой структуре:
```
iam/lines/a01/a01-000u/a01-000u-00.png
          ^^^  ^^^^^^^  ^^^^^^^^^^^^^^
         автор  страница    строка
```

Идентификатор строки `a01-000u-00` кодирует всю структуру:

```python
parts  = line_id.split("-")         # ["a01", "000u", "00"]
writer = parts[0]                   # "a01"
form   = f"{parts[0]}-{parts[1]}"  # "a01-000u"
fname  = f"{line_id}.png"          # "a01-000u-00.png"
return lines_dir / writer / form / fname
```

---

### `run_tesseract()` — распознавание одной строки

```python
def run_tesseract(image_path: Path) -> str:
    img  = Image.open(image_path).convert("RGB")
    text = pytesseract.image_to_string(img, config=TESS_CONFIG)
    return text.strip()
```

`.convert("RGB")` убирает альфа-канал из PNG-файлов (формат RGBA). Tesseract иногда ведёт себя непредсказуемо с изображениями, имеющими прозрачность.

`image_to_string` запускает Tesseract как внешний процесс и возвращает строку. `.strip()` убирает символ `\n`, который Tesseract всегда добавляет в конец вывода.

---

### `main()` — основной цикл

```python
references = []
hypotheses = []
```

Списки для накопления всех референсов (правильных текстов) и гипотез (предсказаний Tesseract). `jiwer` считает CER и WER сразу по всему корпусу — это **не то же самое**, что среднее CER по строкам. Корпусный CER делит суммарное количество ошибок на суммарную длину всех референсов, что является более точной оценкой.

```python
for i, line_id in enumerate(ids):
    if line_id not in gt:
        skipped += 1
        continue
    img_path = line_id_to_image_path(line_id, LINES_DIR)
    if not img_path.exists():
        skipped += 1
        continue
```

Две проверки перед обработкой: есть ли транскрипция в GT (строки со статусом `err` были отфильтрованы в `load_gt`) и существует ли файл на диске (на случай неполной распаковки датасета).

```python
ref  = gt[line_id]
hyp  = run_tesseract(img_path)
references.append(ref)
hypotheses.append(hyp)
writer.writerow([line_id, ref, hyp])
```

Каждая пара (референс, гипотеза) сохраняется в CSV — это позволяет позже проанализировать конкретные ошибки, а не только итоговые числа.

```python
print(f"CER: {cer(references, hypotheses):.4f}")
print(f"WER: {wer(references, hypotheses):.4f}")
```

**CER (Character Error Rate)** считается как `(вставки + удаления + замены) / длина_правильной_строки`. **WER** — то же самое на уровне слов. WER всегда выше CER, потому что одна ошибка в символе делает всё слово неверным.
