#  MNIST Neural Network

Распознавание рукописных цифр на основе базы MNIST с помощью свёрточной нейросети (CNN).

---

##  MNISTDataset

Загружает картинки рукописных цифр из базы MNIST и превращает их в тензоры, готовые для подачи в модель.


---

##  MNISTModel

Принимает на вход тензор `[B, 1, 28, 28]` (батч чёрно-белых картинок 28×28) и возвращает тензор `[B, 10]` с оценками для каждой цифры (0-9). Чем больше число — тем вероятнее эта цифра.

---

###  `conv_block1` — Первый свёрточный блок

**Ищет простые геометрические формы: линии, углы, края.**

| Слой | Что делает                                      | Зачем                                                                                         |
|------|-------------------------------------------------|-----------------------------------------------------------------------------------------------|
| `Conv2d(1, 32, 3, padding=1)` | Применяет 32 фильтра 3×3 к картинке             | Находит простые узоры                                                                         |
| `BatchNorm2d(32)` | Нормализует значения в каждом канале            | Ускоряет обучение, делает его стабильнее                                                      |
| `ReLU(inplace=True)` | Обнуляет отрицательные значения                 | Добавляет нелинейность: без неё все слои свелись бы к одному линейному преобразованию         |
| `Conv2d(32, 32, 3, padding=1)` | Ещё 32 фильтра, смотрит на результаты первых    | Комбинирует простые линии в более сложные формы                                               |
| `BatchNorm2d(32)` | Нормализует значения в каждом канале            | Ускоряет обучение, делает его стабильнее                                           |
| `ReLU(inplace=True)` | Обнуляет отрицательные значения                 | Нелинейность                                                      |
| `MaxPool2d(2)` | Берёт максимум в окне 2×2, сдвигается с шагом 2 | Уменьшает картинку с 28×28 до 14×14, оставляет только самые сильные признаки, экономит память |
| `Dropout2d(0.25)` | Рандомно обнуляет 25% каналов при обучении      | Защита от переобучения                                                                        |

---

###  `conv_block2` — Второй свёрточный блок

**Собирает простые формы в части цифр: петли, перекладины, изгибы.**

| Слой | Что делает | Зачем                                                                           |
|------|-----------|---------------------------------------------------------------------------------|
| `Conv2d(32, 64, 3, padding=1)` | Применяет 64 фильтра к признакам из первого блока | Ищет более сложные узоры                                                        |
| `BatchNorm2d(64)` | Нормализует 64 канала | Ускорение и стабилизация                                                        |
| `ReLU(inplace=True)` | Обнуляет отрицательные значения | Нелинейность                                                                    |
| `Conv2d(64, 64, 3, padding=1)` | Второй слой из 64 фильтров | Собирает части цифр в почти целые фрагменты                                     |
| `BatchNorm2d(64)` | Нормализует значения | Ускорение и стабилизация                                                        |
| `ReLU(inplace=True)` | Обнуляет отрицательные значения | Нелинейность                                                                    |
| `MaxPool2d(2)` | Сжимает окном 2×2 | Уменьшает с 14×14 до 7×7, оставляет только ключевые признаки для классификатора |
| `Dropout2d(0.25)` | Обнуляет 25% каналов при обучении | Защита от переобучения                                                          |

---

###  `classifier` — Классификатор

**Принимает итоговое решение, какая это цифра.**

| Слой | Что делает                                                                | Зачем                                                                         |
|------|---------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| `Flatten(start_dim=1)` | Вытягивает трёхмерный тензор `[B, 64, 7, 7]` в одномерный `[B, 3136]`     | Для дальнейшей работы                                                         |
| `Linear(3136, 256)` | Полносвязный слой: каждый из 3136 входов соединён с каждым из 256 выходов | Анализирует все найденные признаки вместе, взвешивает их важность             |
| `BatchNorm1d(256)` | Нормализует 256 значений                                                  | Ускоряет сходимость, не даёт отдельным нейронам вносить слишком большой вклад |
| `ReLU(inplace=True)` | Обнуляет отрицательные значения                                           | Нелинейность                                                                  |
| `Dropout(0.5)` | Рандомно выключает 50% нейронов при обучении                              | Снижает риск переобучения                                                     |
| `Linear(256, 10)` | Финальный слой: 256 входов → 10 выходов (по одному на цифру 0-9)          | Выдаёт 10 чисел. Чем больше число — тем вероятнее эта цифра                   |
